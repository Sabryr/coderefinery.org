Course goals

The aim of this course is to demonstrate and familiarize the students with best practices and tools in modern research software development. The main focus is on professional tools for developing and maintaining research software efficiently. Since most research codes are developed in a collaborative setting, we will discuss tools and workflows which facilitate this process. But also for a single-person development the demonstrated tools will prove invaluable.

Who the course is for

If you are writing code that is used in research, then this course is for you. If you develop research code and you know all the tools already, become a helper! It's fun!

Prerequisites

  • Basic knowledge of the Linux/Unix shell.
  • Basics in at least one programming language.
  • You will need to bring a laptop.

What we will teach

  • Distributed version control with Git
  • Collaborative web-based development with GitHub
  • Git workflows and release preparation
  • Working with the Jupyter notebook
  • Modular programming
  • Automated testing
  • Continuous integration
  • DevOps, automation, and cloud services
  • Building portable software with CMake
  • Modern code documentation
  • Profiling toolbox
  • Debugging toolbox
