Course goals

The aim of this course is to demonstrate to and familiarize the students with best practices and tools in modern research software development. The main focus is on professional tools for efficiently developing and maintaining research software. Since most research code is developed in a collaborative setting, we will discuss tools and workflows which facilitate this process. Most of the content is also relevant to a single developer.

Format

Hands-on informal and interactive three-day event with type-along type of presentations, live coding and demos. Short tutorials alternate with practical exercises.

What we will teach

  • Collaborative distributed version control with Git
  • Git workflows and release preparation
  • Working with Jupyter notebooks
  • Modular programming
  • Automated testing and continuous integration
  • DevOps, automation, and cloud services
  • Modern code documentation
  • Profiling
  • Debugging

Location

The event is organised at the CSC Training Facilities (room: Dogmi) located in the premises of CSC at Keilaranta 14, Espoo, Finland. The best way to reach us is by public transportation; more detailed travel tips are available.

Who the course is for

If you are writing code that is used in research, then this course is for you.

If you develop research code and you know all the tools already, join us as a helper! It's fun, and you always learn something new about a subject by teaching it.

Prerequisites

  • Basic knowledge of the Linux/Unix shell.
  • Basics in at least one programming language.
  • You will need to bring a laptop.
